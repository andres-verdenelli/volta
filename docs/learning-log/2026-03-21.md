## Qué aprendí hoy

### 1. `layout.tsx` no es para todo

El layout global sirve para estructura compartida de toda la app.
No conviene meter ahí decisiones visuales específicas de una sola pantalla, como centrar exclusivamente auth, salvo que después se arme un layout específico para esa sección.

### 2. Formularios: ser explícito ayuda

Para formularios básicos conviene usar:

- `Label` con `htmlFor`
- `Input` con `id`
- `name` en los campos
- `type` cuando aporta semántica o comportamiento

### 3. Repetir un poco no es pecado

Repetir una estructura simple como un bloque `label + input` dentro de la misma página todavía está bien.
No hace falta abstraer demasiado temprano si la repetición sigue siendo clara y fácil de leer.

### 4. `asChild` en `shadcn`

`asChild` permite aplicar el estilo del componente al hijo en lugar de renderizar el elemento por defecto.
Eso es útil cuando se quiere que un `Link` tenga estilo de botón sin perder la semántica de link.

## Decisiones de diseño tomadas

- Mantener el estilo visual del link secundario con `underline`, porque se siente mejor así
- No agregar más separación entre el texto secundario y el link, porque visualmente ya estaba donde querías
- Priorizar consistencia entre `login` y `signup` antes de seguir con otras pantallas
